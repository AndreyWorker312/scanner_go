# Network Scanner Service: Архитектура и документация

## Обзор
Network Scanner — это веб-сервис для сканирования TCP-портов с сохранением результатов в PostgreSQL. Сервис предоставляет REST API для управления сканированиями и просмотра результатов.

## Архитектура системы

### Основные компоненты

1. **Главное приложение** (`cmd/server/main.go`)
    - Инициализирует все компоненты системы
    - Запускает HTTP-сервер
    - Обеспечивает graceful shutdown

2. **Конфигурация** (`config/`)
    - Загружает настройки из `.env` файла
    - Предоставляет структурированную конфигурацию всем компонентам

3. **Сканер портов** (`scanner/`)
    - Реализует логику сканирования TCP-портов
    - Поддерживает диапазоны портов (80-100) и списки (80,443,8080)
    - Имеет настройки таймаутов и повторных попыток

4. **Хранилище данных** (`repository/`)
    - Работает с PostgreSQL
    - Сохраняет историю сканирований и результаты

5. **API обработчики** (`handler/`)
    - Обрабатывает HTTP-запросы
    - Взаимодействует со сканером и хранилищем

## Как работает сканирование портов

1. Пользователь отправляет POST-запрос на `/api/v1/scan` с параметрами:
   ```json
   {
     "ip": "192.168.1.1",
     "ports": "80-100,443,8080"
   }
   ```

2. Сервис парсит список портов:
    - Поддерживаются форматы: одиночный порт (`80`), диапазон (`80-100`), список (`80,443`)

3. Для каждого порта выполняется:
    - Попытка TCP-соединения с таймаутом (по умолчанию 500ms)
    - До 3 повторных попыток (настраивается)
    - Задержка между попытками 1s (настраивается)

4. Результаты сохраняются в БД:
    - Информация о запросе сканирования
    - Статус каждого проверенного порта (открыт/закрыт)

## Полная документация API

### 1. Запуск сканирования

**Endpoint**: `POST /api/v1/scan`

**Параметры**:
```json
{
  "ip": "string, обязательный",
  "ports": "string, обязательный (форматы: 80, 80-100, 80,443,8080)"
}
```

**Пример ответа**:
```json
{
  "request_id": 1,
  "ip": "192.168.1.1",
  "ports": "80-100,443,8080",
  "open_ports": [80, 443],
  "status": "completed"
}
```

### 2. Получение истории сканирований

**Endpoint**: `GET /api/v1/history`

**Параметры**:
- `limit` - ограничение количества результатов (по умолчанию 10)
- `offset` - пагинация

**Пример ответа**:
```json
[
  {
    "id": 1,
    "ip": "192.168.1.1",
    "ports": "80-100,443,8080",
    "open_ports_count": 2,
    "created_at": "2023-10-01T12:00:00Z"
  }
]
```

### 3. Получение детальных результатов

**Endpoint**: `GET /api/v1/results/{id}`

**Пример ответа**:
```json
{
  "request": {
    "id": 1,
    "ip": "192.168.1.1",
    "ports": "80-100,443,8080",
    "created_at": "2023-10-01T12:00:00Z"
  },
  "results": [
    {
      "port": 80,
      "is_open": true,
      "scan_duration_ms": 120
    },
    {
      "port": 443,
      "is_open": true,
      "scan_duration_ms": 80
    }
  ],
  "statistics": {
    "total_ports": 23,
    "open_ports": 2,
    "scan_duration_total_ms": 1500
  }
}
```

## Настройки окружения

Все параметры настраиваются через `.env` файл:

```ini
# Сервер
SERVER_PORT=8006

# База данных
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=network_scanner

# Сканер
SCANNER_TIMEOUT=500ms  # Таймаут проверки порта
SCANNER_MAX_RETRIES=3  # Макс. попыток проверки
SCANNER_RETRY_DELAY=1s # Задержка между попытками
```

## Запуск сервиса

### Локальная разработка

1. Установите PostgreSQL и создайте БД:
   ```bash
   createdb network_scanner
   ```

2. Запустите миграции:
   ```bash
   make migrate-up
   ```

3. Запустите сервер:
   ```bash
   make run
   ```

### Docker-развертывание

1. Соберите и запустите контейнеры:
   ```bash
   make docker-up
   ```

2. Остановите сервис:
   ```bash
   make docker-down
   ```

## Особенности реализации

1. **Многопоточное сканирование**:
    - Каждый порт проверяется в отдельной goroutine
    - Используется sync.WaitGroup для ожидания завершения

2. **Контекст выполнения**:
    - Поддержка отмены операций через context.Context
    - Graceful shutdown сервера

3. **Повторные попытки**:
    - Настраиваемое количество попыток для каждого порта
    - Экспоненциальная задержка между попытками

4. **Безопасность**:
    - Валидация входных параметров
    - Ограничение диапазона портов (1-65535)
